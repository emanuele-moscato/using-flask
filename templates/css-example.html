How to use cascading style sheets (CSS)

Flask automatically looks for (and serves) files from the static/ directory, which has to be put at the same level as the app file. Subdirectories are then possible and can be specified in the path in the link object in the head of the html files that uses the stylesheet.

Browser caching

When we update the CSS stylesheet used by a page, this may not be reflected in what we see in the browser. The reason is called browser caching: the browser keeps the information about the old version of the CSS file in memory and doesn't realise when changes happen. There are two workarounds,

  • Set up headers that tell the browser not to cache information from this website (see a guide here). This can be fairly complicated in the context of a Flask app.
  • Manually deleting the browser's cache after updating the CSS file. For Chrome users, a giude can be found here. For testing purposes, this is sufficient.

Custom elements

We can define style attributes in a CSS file for an element with a specific id. Since ids should be unique (no two elements should have the same id) this means that we are specifying style options for that element alone.
